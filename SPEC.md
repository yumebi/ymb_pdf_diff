# YMB PDF DIFF 仕様設計

## 1. 目的
2つのPDFを読み込み、差異を初心者でも一目で分かる形で表示する。Windows / Mac 両対応。
比較ロジックは「テキスト差分」と「見た目(画像)差分」の両方を扱う。

## 2. 技術スタック
| 用途 | ライブラリ |
|---|---|
| GUI | PySide6 |
| PDF読込・テキスト抽出・画像化 | PyMuPDF (fitz) |
| 画像差分 | Pillow / numpy |
| テキスト整列・差分 | difflib (SequenceMatcher) |
| Excelレポート出力 | openpyxl |
| OCR(スキャンPDF対応) | pytesseract + Tesseract-OCR(外部バイナリ) |
| 配布(ビルド) | PyInstaller (Win exe / Mac app) |
| 配布(インストーラ) | Win: Inno Setup / Mac: dmg(create-dmg等) |
| ウィンドウ状態保存 | PySide6 `QSettings` |
| バージョン比較 | packaging.version |
| HTTP取得(自動更新チェック) | requests |

選定理由: 開発しやすさ優先(軽量さは優先度下)。PyMuPDFがテキスト抽出(座標付き)とページ画像化を1ライブラリで両方カバーできるため実装コストが低い。

## 3. 差分アルゴリズム(ページズレ対策込み)

### 3.1 課題
ページ番号で固定的に比較する(file_A[i] vs file_B[i])と、1文の増減でページ構成がズレ、以降の全ページが「差分あり」と誤判定される。

### 3.2 方式: 全文アライメント
0. テキストレイヤーがないページ(スキャンPDF)は、PyMuPDFのテキスト抽出結果が空の場合に `pytesseract` でOCRし、テキスト化してから以降の処理に合流させる(OCR言語は `jpn+eng` 固定)
1. 各PDFのページからテキストを行単位で抽出し、各行に元ページ番号タグを付与
2. file_A・file_Bそれぞれの行シーケンスを結合
3. `difflib.SequenceMatcher` で2つの行シーケンス全体を整列し、opcode(`equal` / `replace` / `insert` / `delete`)を取得
4. 各opcodeブロックが参照する行から、両ファイルの該当ページ番号区間を算出
5. ページ対応表(file_A page ⇔ file_B page)を作成
   - `equal` ブロックのみで構成されるページ → 「差分なし」(物理ページ番号がズレていても無視)
   - `replace` / `insert` / `delete` を含むページ → 「差分あり」、対応するページ番号同士のみ詳細比較対象にする
   - 対応するページ番号がズレている場合は常に「ページ移動」フラグを立てる(閾値なし、例: A5ページ→B7ページ、内容自体は変更なし)。差分カウントには含めず参考情報として表示

### 3.3 見た目(画像)差分
ページ対応表で決まったペアのみ画像比較する(単純な同index比較はしない)。
- ページ画像をPyMuPDFでpixmap化
- `ImageChops.difference` 等でピクセル差分検出 → 差分領域を矩形でハイライト

### 3.4 効果とトレードオフ
- 効果: 1文挿入によるページズレ起因の誤検出をほぼ解消
- トレードオフ: 単純index比較より実装複雑、文書全体アライメント分の処理コストが増える

## 4. 出力機能

### 4.1 セッション保存(再利用用)
比較結果をアプリ内で再読込できる形式で保存する(再計算せず差分一覧をすぐ復元する用途)。
- 保存内容: ファイル情報、ページ対応表(ページ移動フラグ含む)、差分一覧(テキスト差異・座標)、差分キャプチャ画像
- 形式: 専用拡張子(zip構造)、中に `result.json`(メタ情報・差分一覧・ページ対応表) + `captures/*.png`(差分キャプチャ画像)を同梱。自己完結のため移動・配布しても壊れない

### 4.2 Excelレポート出力(openpyxl)
- **1シート目「サマリー」**
  - ファイル名(A/B)、比較日時、総ページ数、総差分数
  - ページ一覧テーブル(ページ番号A/B、差分有無、差分件数)+ 各ページ詳細シートへのハイパーリンク
- **ページ単位シート**(差分のあるページごとに1シート、シート名はページ番号ベースで命名・31文字制限とExcel禁止文字を考慮しサニタイズ)
  - 該当ページの前後キャプチャ画像(差分領域ハイライト付き)を埋め込み
  - テキスト差異リスト(変更前/変更後を列で対比、追加=緑・削除=赤・変更=黄の背景色分け)

## 7. 追加機能(実装予定)

### 7.1 マルチモニタ境界チェック(ウィンドウ位置復元)
- 終了時にウィンドウの位置・サイズを`QSettings`に保存、起動時に復元
- 復元時、`QGuiApplication.screens()` 各画面の `availableGeometry()` と比較し、保存位置がどの画面にも収まらない場合は最も近い画面範囲内にクランプ(外部モニタ取り外し後に画面外へ消える事故を防止)

### 7.2 自動更新チェック
- 配布物にバージョン番号を埋め込み(`__version__`)
- 起動時に `version.json` を取得、`packaging.version` で現在バージョンと比較
- 新バージョンあればステータスバーに通知(例:「新しいバージョン v1.2.0 があります」+ダウンロードページへのリンク)。自動インストールはせず通知のみ
- 設置場所・チェック頻度は6章で確定済み
